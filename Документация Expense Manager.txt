Официальная Документация FinancialTrackerApp
Введение
Данная документация предназначена для ознакомления с функциональностью и использованием приложения FinancialTrackerApp, разработанного для управления и отслеживания финансовых транзакций. Приложение позволяет пользователям вводить, редактировать и удалять данные о транзакциях, а также генерировать сводные финансовые отчёты.

Системные требования
Операционная система: Windows 7 или выше, MacOS X 10.8 или выше, Linux
Java Runtime Environment (JRE): версия 8 или выше

Конфигурация приложения
Скачайте и установите Java Development Kit (JDK) и настройте переменную среды JAVA_HOME.
Откройте файл FinancialTrackerApp.java и скомпилируйте его с помощью JDK.
Запустите скомпилированный файл, чтобы открыть приложение.
Руководство пользователя
Добавление транзакции
Для добавления новой транзакции выполните следующие шаги:

Введите название транзакции в текстовое поле Name.
Укажите количество в текстовом поле Amount.
Введите цену в текстовое поле Price.
Выберите тип транзакции (Expense для расхода или Income для дохода) из выпадающего списка Type.
Нажмите кнопку Add для добавления транзакции в список и базу данных.
Редактирование и удаление транзакций
Для редактирования или удаления транзакции:

Выберите транзакцию из списка слева.
Для редактирования измените данные в текстовых полях и нажмите Edit.
Для удаления нажмите Delete, и транзакция будет удалена из списка и базы данных.
Генерация отчёта
Для создания финансового отчёта:

Нажмите кнопку Create Report.
Отчёт будет автоматически сгенерирован и отображён в центральном текстовом поле.
Архитектура приложения
Класс Item
Класс Item инкапсулирует данные о транзакции и предоставляет методы для доступа к этим данным.

Класс FinancialTrackerApp
Класс FinancialTrackerApp является основным классом приложения, который реализует пользовательский интерфейс и логику взаимодействия.
Безопасность и конфиденциальность
Приложение FinancialTrackerApp разработано с учётом принципов безопасности и конфиденциальности данных. Все данные хранятся локально и не передаются третьим лицам.
